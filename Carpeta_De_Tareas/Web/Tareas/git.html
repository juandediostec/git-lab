Flujo de Trabajo Básico (Para proyectos pequeños o personales)

  1. Clonar el repositorio.
  2. Hacer los cambios en el código.
  3. Ver estado de los cambios.
  4. Agregar los cambios al área de preparación.
  5. Hacer un commit.
  6. Subir los cambios al repositorio remoto.

Flujo de Trabajo con Ramas (Para proyectos colaborativos)

  1. Clonar el repositorio.
  2. Crear una nueva rama
  3. Hacer cambios en el código
  4. Ver estado de los cambios.
  5. Agregar los cambios al área de preparación.
  6. Hacer un commit.
  7. Subir la rama al repositorio remoto.
  8. Crear un Pull Request en GitHub.
  9. Actualizar tu rama local después del merge

Tabla de Comandos de Git

¿Qué hace el comando? Comando
Clonar el repositorio git clone https://github.com/usuario/repositorio.git
Ver estado de los cambios git status
Agregar los cambios al área de preparación git add .
Hacer un commit git commit -m "Mensaje descriptivo del cambio"
Subir los cambios al repositorio remoto git push origin main
Crear una nueva rama git checkout -b nombre-de-la-rama
Subir la rama al repositorio remoto git push origin nombre-de-la-rama
Actualizar la rama local después del merge git pull origin main
